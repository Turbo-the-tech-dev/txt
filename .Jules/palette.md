## 2026-02-20 - [Stable Copy Buttons and Independent Feedback]
**Learning:** In documentation layouts using `overflow-x: auto` for code blocks (`pre`), absolutely positioned children like 'Copy' buttons will scroll horizontally with the content if they are inside the scrolling element. Wrapping the `pre` in a `position: relative` container and making the button a sibling of the `pre` keeps the button stably positioned in the corner. Additionally, storing timeout IDs locally within a closure (per-button) prevents race conditions where rapid clicks on different buttons interfere with each other's feedback states.
**Action:** Always wrap scrolling code blocks in a non-scrolling relative container when adding fixed UI overlays, and use local closures for element-specific feedback timers.
