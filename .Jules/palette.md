## 2026-03-02 - [Accessibility & Security Foundations]
**Learning:** For static documentation sites that render Markdown client-side, the foundation of good UX includes keyboard-accessible navigation (Skip links), high-contrast badges (WCAG AA), and robust client-side sanitization (DOMPurify) to prevent XSS from fetched content.
**Action:** Always include a Skip to Content link and integrate DOMPurify when using 'marked' for Markdown rendering in future static sites.
